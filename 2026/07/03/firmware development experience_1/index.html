AWR2243固件开发学习经验_1

一. 研究背景

仅以此系列文章记录个人在摸索学习AWR2243级联雷达时所涉及的一些经验，供后续回顾复习以及他人学习交流。本系列仅针对如何从原本mmWavestudio的脚本控制转向嵌入式固件运行，从而适应各种独立部署环境。

1.1 设备组成

设备整体由MMWCAS-DSP-EVM与MMWCAS-RF-EVM两块板组成前者用来处理射频板的雷达数据信号，给射频板输入5V供电，可以进行信号处理，比如FFT、CFAR等。其搭载的是TDA2x处理器，而不是带视觉ISP的TDA2Px，这对后面的程序设定有帮助。
DSP板我在当前开发中主要用到的接口有：Ethernet Port（网口）、UART/Terminal（Mini USB(mini-B)）、Pwr Input（电源线）

图 1：DSP板正面图

从下图的功能原理框图可以看到，60-Pin MIPI Debug Header与TDA2x板的JTAG口直接连接，因此主板上并没有集成仿真器，我也并没有MIPI-60的线，因此并不能直接使用CCS联机调试（JTAG烧录），只能用SD卡启动，不过也好，一块SD卡也不贵，省了不少钱。

图 2：功能原理框图

了解了基本结构后，我们便可以开始着手从TI的官方SDK里开始学习开发了。

1.2 功能

在先前的lua脚本中，我们已经实现了脉冲的配置与采集数据保存的功能，并且后续的数据处理采用了matlab，可以绘制出距离-方位图、距离-速度图、速度-时间图、目标点云图等，但先不提采集数据与处理数据并不同步的缺点，连采集数据都必须连接电脑，哪怕使用了延迟脚本，雷达依然需要开着mmWavestudio，十分不方便，这些功能最好都集成到DSP板上。
脉冲配置与数据处理可以参考：TI毫米波级联雷达评估板 MMWCAS-DSP-EVM 和MMWCAS-RF-EVM
任务流应该为：ARM核心负责主控程序，初始化4个雷达芯片，并启动级联触发；TDA2接收射频板通过CSI-2传入的高速数据流，并转运至DDR内存；接收到数据后DSP便可以通过触发中断开始各种FFT以及CFAR目标检测等，后续也可以生成目标点云图。最终处理后的数据可以通过以太网，将点云坐标或热力图矩阵传至上位机。

二. 开始动手

2.1 开发环境

首先下载CCS(Code Composer Studio)，我用的是CCS 21.0.0，该版本已经抛弃了原本的Eclipse架构，现在长的跟VScode一样，所以网上有些教程过时了，不过影响主要是一些基本操作，比如工作空间设置还有工程导入那些，问题不大。

接着，对于TDA2xx系列芯片，需要从官网下载PROCESSOR-SDK-RADAR,选择Installer Package for Windows的那个。安装时直接放在默认路径C:\PROCESSOR_SDK_RADAR_03_08_00_00，里面的代码往往写得很死，后面我也会提到这一点的例子。

此外还需要Tera Term进行串口监听，用USB将TDA2板子与电脑相连后，选择对应的COM口，设置-串口-比特率115200-New setting。

安装gcc-arm-none-eabi，在这里下载，如果下载的是4.9.3版本，则不需要管下面的操作；如果下载的是后续版本，比如我的5.4.1，则需要复制一份欺骗一下TI的程序，因为程序写死了要用4.9.3

sbl (Secondary Boot Loader) 会MLO文件，相当于电脑主板的 BIOS，负责在开机瞬间把内存跑通，并把 AppImage 从 SD 卡里拉起来。
在文件管理器里找到

可以看到4个文件夹，分别是opp_high、opp_low、opp_nom、opp_od，这里是由于TI存在动态电压与频率调节，TI编译一次会生成不同性能模式的引导程序。
opp_nom (Nominal - 标称模式)： 这是芯片的标准运行频率和电压。该版本比较稳定。
opp_high (High - 高性能模式)： 芯片处于超频或满血状态，DSP 和 ARM 核心跑在最高频率。性能最强，但功耗最大，主板发热也会增加。
opp_od (OverDrive - 极速/过载模式)： 介于 Nominal 和 High 之间的一种特殊压榨性能模式。
opp_low (Low - 低功耗模式)： 降频运行，省电，但可能处理不了 4 芯片级联的数据吞吐量。
显然我们使用nom标称模式就行了。打开opp_nom\tda2xx-evm-radar找到MLO即可，注意先不要放进SD卡里，复制到桌面备用。

2.4 保存与上电

准备一张SD卡，将其格式化为FAT32格式，如果空间大于32G，其格式可能是exFAT，这时要用DiskGenius来进行格式化。DiskGenius可以在图吧工具箱里找到。注意格式化时要先删除当前分区，保存后，再新建分区。
考虑到TI 的 Boot ROM可能比较死板，MLO需要在扇区的最前面，SD卡格式完毕后，先将MLO复制进去，再复制AppImage进去。
接着要将雷达调整为SD卡模式。如果你先前就已经用mmWavestudio成功让雷达跑起来过，则不需要调整User Boot SW。

到这里我们一整条流程走完了，接下来就要上电启动试试看了！将雷达电源接上，USB线连接，由于暂时没有写信号处理与保存，所以可以不接网线。把准备好的SD卡从读卡器上拿下来，插进DSP板的卡槽，注意先前配制好端口与波特率的Tera Term，可以看到雷达的启动日志，并且会给出两个功能选项，展示结果应该包含以下内容：
